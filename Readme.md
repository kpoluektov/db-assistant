Yandex AI Studio. Пример использования OpenAI Agents SDK для построения ассистента 

В данном сценарии решается прикладная задача - создание ассистента-помощника работы с БД. демонстрируется сценарий делегирования (Handoff), который позволяет разделить агентов по решаемым задачам.
### MetadataAgent
- анализ метаданных: таблицы, колонки, статистика, индексы. На основании этих данных может выдавать базовые рекомендации по оптимизации SQL запросов;
### DataMaskingAgent
- выдает рекомендации по маскированию данных исходя из описания таблиц, которые ищет в контексте диалога (agents.SQLiteSession). Выдает их на основании файлов, загруженных в Yandex AI Studio vector_store

Для работы нужно 
- склонировать репозиторий
- запустить сборку 
```
docker-compose build
```
- создать .env с описанием подключения к базе c примерным содержимым (пример подключения к Managed Service for MySQL)

```
META_URL=http://web:8080
MDATA_HOST=mysql-fqdn.mdb.yandexcloud.net
MDATA_PORT=3306
MDATA_USER=<user>
MDATA_PASS=<password>
MDATA_TYPE=mysql
MDATA_BASE=pol
MCP_PORT=8081
MSP_HOST=mcp
SSE_MODE=true
MDATA_CAPATH=/app/root.crt
```
- создать .env.agent 
```
YANDEX__FOLDER_ID=<folder_id>
YANDEX__AUTH=<API-KEY>
YANDEX__ASSISTANT_INSTRUCTION=Ты главный ассистент безопасности. твоя задача комплексно отвечать на  запросы используя своих  агентов.Если обращение содержит имя схемы и имя таблицы, то всегда делегируй задачу  агенту MetadataAgent. Он умеет собирать метаданные по объектам хранения СУБД. Потом передавай полученную информацию агенту DataMaskingAgent - он на основании собственных знаний дополнит ответ правилами обезличивания.В режиме диалога используй предыдущие ответы. Ответы возвращай в формате html - только контент  без кавычек и тегов html и  body. 
YANDEX__METADATA_INSTRUCTION=Ты ассистент работы с данным, анализируешь метаданные таблиц, оптимизируешься sql на основе описания таблиц, индексов и статистики; для поиска оптимального плана исполнения используй mcp сервер.С помощью него анализируй структуру таблиц, количество записей, наличие необходимых индексов.Через MCP сервер проверяй фактическое значение параметров базы данных.Одним из вариантов рекомендаций может быть переписывание текста запрос, но в этом случае нужны рекомендации, которые не повлияют на ResultSet.Не давай рекомендаций проверки параметров, которые не нужно менять - проверяй их фактическое значение в базе сам.Только после полного анализа выдай актуальные рекомендации.Ответы возвращай в формате html, только контент  без кавычек и тегов html и  body. Не добавляй  markdown разметку в ответ
YANDEX__MASKING_INSTRUCTION=Ты ассистент аналитик безопасной обработки данных. Исходя из вопроса и контекста диалога выдавай рекомендации, строго следуя индексу RAG.В ответах цитируй источник рекомендаций. Ответы возвращай в формате html
YANDEX__URL=https://ai.api.cloud.yandex.net/v1
YANDEX__MODEL=gpt://${YANDEX__FOLDER_ID}/gpt-oss-120b/latest
YANDEX__GET_INFO_MCP_URL=http://mcp:8081/sse
YANDEX__LOG_FILE_NAME=agent_main.log
YANDEX__SECRET_KEY=my_very_secret_key
YANDEX__MASKING_INDEX_ID=<vector_store_id>
YANDEX__PORT=8083
```
Запустить ассистента
```
docker-compose up -d
```
